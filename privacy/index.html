Privacy Policy for GeoVitals

Effective Date: June 15, 2026

GeoVitals does not collect, store, or share any personal data.

The extension does not track browsing history, keystrokes, form data, or any user activity beyond what is described below.

Data Fetched

When you hover a country name, the extension fetches publicly available information from the following trusted sources solely to display it in a tooltip:

  • jsdelivr CDN (cdn.jsdelivr.net) — country demographics data, passport visa index, and daily currency exchange rates
  • Google News RSS (news.google.com) — recent news headlines for the hovered country
  • disease.sh — COVID-19 statistics
  • World Health Organization (who.int) — disease outbreak alerts

All fetched data is cached locally in memory for up to 30 minutes to minimise network requests. No fetched data is stored permanently or transmitted to any server owned by this extension.

Passport Country Detection

To show visa requirements, the extension reads your browser's accepted languages (via the chrome.i18n.getAcceptLanguages API) to infer your country on first install. This information is stored locally in Chrome's sync storage (chrome.storage.sync) and is only used to determine visa requirements for the hovered country. It is never transmitted to any external server owned by this extension.

You can change or clear your passport country at any time from the Options page.

Local Storage

The extension stores the following data locally using Chrome's sync storage:

  • Extension enabled/disabled state
  • Your settings (show/hide visa, news, COVID, WHO data)
  • Your passport country (ISO2 code and name)
  • The last hovered country and its data (for display in the popup)
  • Country exclusion list

This data never leaves your browser except as part of Chrome's standard sync mechanism if you have Chrome Sync enabled.

No Third-Party Tracking

No data is ever sent to any third-party server owned by this extension. GeoVitals contains no analytics, no advertising, and no tracking of any kind.

Contact

If you have any questions about this privacy policy, please open an issue on the GitHub repository.
